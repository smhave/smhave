SSL

Secure Sockets Layer, or SSL for short, is an Internet protocol that adds a layer of security to Hypertext Transfer Protocol (https). Whether or not a website has this added layer of security is to look at the beginning of the URL in the address bar. If it starts with https://, then the s after the http denotes it is secure with SSL. if the URL starts with http://, without the s, then it is not secure with SSL, and is more vulnerable to an online attacker.

I did not know ISP’s can inject ads and other content on other people’s web pages. When she was talking about the identity certificates, it sounded like HTTPS makes it so the browser and user can be sure that the user isn’t being phished from a fake site.

Apparently HTTPS can make it so eavesdropping on the network can’t see the communications between client and server. I’m wondering if this makes public wi-fi secure enough. When I first learned about man-in-the-middle attacks years ago, that scared me off of doing anything that required logging in to something on public wifi completely. I am now wondering if HTTPS kind of takes away that danger, at least on sites that have HTTPS.

One of the myths she talked about in the video is that if a site doesn’t have users logging in or entering in private information, that HTTPS is not needed. I was also of this opinion. But she explained why this is not true, with the example that other people’s (the ISP’s, maybe?) obtrusive ads can be injected to the site, and trick your users into downloading malware. That is a huge reason to be using HTTPS.

The multiplexing in HTTP/2 seemed really cool to me. Really good way to speed up network performance, instead of having to do request/response, request/response, request/response the whole time. Same with the server push for style sheets and such when requesting HTML pages.
