Chrome DevTools

Google's Chrome browser comes with a web development tool called DevTools. DevTools makes analyzing the front-end coding of websites easier than just viewing the whole page in a code editor or IDE . You can use DevTools to preview what would happen if you change or delete certain parts of the code that are already coded in. You can also more easily see exactly which block of code is affecting which element on the page.

I used DevTools last semester in Rio’s CIT 93 Client Side Scripting class. Though I didn’t really use anything but the Console tab. The video I found was very informative about the Elements tab. Some of the various things I learned from the video:

When you hover over an HTML element that has children, it gets highlighted on the page. Conversely, you can highlight something on the page and it will take you to the HTML code for that element.

Double click on the text in DevTools, change it, and it immediately changes it on the page (on your browser only, not to the server).

Add elements by “add attribute” function.

Click the “mobile view” button to see what the page looks like on a mobile device. Select between views of many different mobile devices.

“Network: Configure throttling” Test with various Internet speeds.

In the CSS pain, something struck through could mean it’s being overridden by another attribute. On a color hex code, you can click a little box that brings up a color palette you can choose a specific shade from. Open up children on an attribute, see all kind of different more specific options for similar attributes. Click the “computed” tab it alphabetically lists all the styles that are currently being applied.

Click the “event listeners” tab for JavaScript event listeners.
